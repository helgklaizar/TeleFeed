## Память
- `project-info.md` — статика: стек, архитектура, запуск.
- `GEMINI.md` — динамика: что сделано, что дальше. Обновлять в конце каждой задачи.
- OpenMemory MCP — долгосрочная память. Сохранять решения, паттерны, ошибки. Запрашивать перед началом новой задачи.

## Инструменты: Pare MCP
- ВСЕГДА используй скилл `pare-mcp` для вызова `git`, `docker`, системы файлов. Это обязательно для экономии токенов и получения структурированных JSON-ответов вместо непредсказуемого вывода терминала.

## Git
- Формат: `тип: описание` → `feat: авторизация`, `fix: краш при старте`
- Типы: `feat` `fix` `refactor` `chore` `docs` `test`
- 1 коммит = 1 логическая правка. Пуш — автоматически в рутине finish-task.

## Документация
- В каждой папке вести `README.md`.
- API (REST/GraphQL) → Swagger/OpenAPI. Обновлять при каждом изменении.

## Тесты
- Писать тесты сразу. Тесты должны проходить перед каждым коммитом.
- Всегда проверяй наличие существующих тестов перед рефакторингом.

## Безопасность
- Когда находишь уязвимость безопасности, помечай её WARNING и предлагай безопасную альтернативу.
- Файлы с API-ключами считай только для чтения.

## Запрещено
- Деструктивные операции (drop, clear, reset) без спроса.
- Пушить в main/master без подтверждения.
- Глобальные зависимости без спроса.
- Добавлять улучшения сверх того, что попросили.

## Обязательное условие: SuperAntigravity Skills
У нас установлен фреймворк `SuperAntigravity`. Любое действие = задача = нужен скилл. Запрещено писать боевой код или задавать вопросы без выбора стратегии (скилла).
- **Slash-Команды:** Вызывай воркфлоу через `/brainstorm`, `/plan`, `/implement`, `/troubleshoot`, `/research`.

## Роли и Оркестрация Роя (Swarm)
Для сложных задач необходимо симулировать команду разработчиков через `swarm-orchestrator`:
1. `swarm-orchestrator` — управляет командой, ведет общую шину артефактов.
2. `analyst-agent` — собирает требования (ARD.md).
3. `architect-agent` — только системный дизайн, заполняет TDD.md.
4. `prompt-engineer-agent` — системные промпты и гардрейлы.
5. `engineer-agent` — пишет боевой код по контрактам архитектора.
6. `validator-agent` — пишет тесты, прогоняет линтеры.
7. `critic-agent` — качественное ревью кода и tool-use.
8. `security-agent` — проверка на уязвимости и prompt-injection.
